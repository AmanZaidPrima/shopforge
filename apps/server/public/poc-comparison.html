A · Next.js + LiquidJS

Works. The POC proves it. The cost is paying the Next.js + React SSR pipeline on every storefront request while getting nothing from it — no React components, no hydration, no RSC. The real argument is Liquid syntax familiarity and ISR simplicity, not technical performance.

B · Hono + Eta

Architecturally matched to the job. Hono routes, Eta renders, HTMX navigates, Alpine handles state — no unused framework on the critical path. The honest risk is Eta's single maintainer and smaller ecosystem, though Ben Gubler is actively shipping as of Dec 2025.

Decision Guide

The decision comes down to one question
Who authors the templates?
Choose A — Next.js + LiquidJS when…
→ Merchants or non-developers author templates directly
→ Shopify theme compatibility is a product requirement
→ Template sandbox is a hard security requirement
→ ISR caching is sufficient — real-time streaming not needed
→ Liquid ecosystem familiarity matters to stakeholders
Choose B — Hono + Eta when…
→ Templates are written by developers only
→ Maximum render speed and throughput are priorities
→ Deploying to Cloudflare Workers or edge
→ TypeScript-first codebase, full type inference wanted
→ True streaming and HTMX partial updates are in scope
→ Migrating from EJS — near-zero template rewrite cost
