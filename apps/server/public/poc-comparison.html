Why this is one architecture, not bolted-on pieces
In B, every layer serves the same goal: server renders HTML, TransformStream delivers it progressively, HTMX requests more server-rendered HTML for navigation, Alpine adds local interactivity — all sharing one render path. In A, Next.js runs a React SSR pipeline that produces nothing, ISR caches pages that break on dynamic data, and there's no navigation mechanism without React components the storefront doesn't use.

Architecture Decision

Next.js is a React framework. Our storefront uses zero React on the shopper path.

Verdict

Chosen stack
Hono + Eta
on Cloudflare Workers
Every metric — TTFB, throughput, JS weight, cold start — points the same direction. More importantly, the pieces form a coherent architecture: Hono routes and renders, Eta compiles once and executes in microseconds, TransformStream delivers HTML progressively, HTMX navigates by requesting more server-rendered HTML from the same handler, Alpine adds interactivity without hydration. No layer runs unused, no pipeline cost is wasted.
Known risks — acknowledged and mitigated
!
Single maintainer (bgub) — Actively shipping as of Jan 2026. MIT-licensed; fork is a one-day operation if needed.
!
No Liquid syntax — Templates are developer-authored. Merchants configure sections via the visual editor, not by writing templates.
!
Eta not sandboxed — Templates are platform-controlled. If merchant authoring is added later, isolated-vm solves it.
